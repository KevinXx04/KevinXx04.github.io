Backend Funcavida

Descripción del Proyecto

El proyecto implementa una API RESTful con arquitectura en capas (MVC + Service/Repository), integrando autenticación JWT, gestión de permisos basada en roles, y un sistema completo de recuperación de contraseñas por email. Se trabajó en equipo siguiendo metodologías ágiles y prácticas de desarrollo colaborativo con control de versiones.

Arquitectura del Proyecto

Arquitectura backend basada en Node.js con patrón MVC (Modelo-Vista-Controlador), implementando APIs RESTful para la comunicación con el frontend y servicios externos:

  • API RESTful con Express.js para manejo de rutas y controladores
  • Servicio de envío de correos electrónicos con Nodemailer para recuperación de contraseñas
  • Base de datos para gestión y persistencia de datos con Prisma ORM y MySQL
  • Sistema de autenticación y autorización de usuarios con JWT y bcrypt
  • Integración con módulos de gestión de la ONG:
    • Gestión de Usuarios, Roles y Permisos (RBAC - Control de acceso basado en roles)
    • Gestión de Sobrevivientes, Padrinos y Voluntarios (Con relaciones complejas)
    • Gestión de Actividades, Sedes y Activos (Sistema completo de eventos y recursos)
    • Gestión de Proveedores y Categorías (Control de inventario y clasificación)
    • Sistema de Contactos de Emergencia (Para todos los participantes)
    • Bitácoras de Seguridad (Audit logs para todas las operaciones críticas)
    • Recuperación de Contraseñas (Sistema completo con tokens seguros y expiración)
    • Estadísticas y Reportes (Dashboard con métricas clave de la organización)

Tecnologías Aplicadas

  • Node.js - Entorno de ejecución de JavaScript del lado del servidor
  • Express.js - Framework web minimalista y flexible para Node.js
  • Nodemailer - Biblioteca para envío de correos electrónicos (recuperación de contraseñas)
  • JavaScript - Lenguaje de programación principal del backend
  • Prisma ORM - ORM moderno para Node.js con TypeScript support
  • MySQL - Sistema de gestión de base de datos relacional
  • JWT (JSON Web Tokens) - Autenticación y autorización segura sin estado
  • bcrypt - Hash de contraseñas con algoritmo de encriptación fuerte
  • Git - Control de versiones distribuido para desarrollo colaborativo
